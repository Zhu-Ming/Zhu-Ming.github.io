LeetCode-24两两交换链表中的结点

2019-11-15
LeetCode

题目：

给定一个链表，两两交换其中相邻的节点，并返回交换后的链表。

你不能只是单纯的改变节点内部的值，而是需要实际的进行节点交换。
示例:

给定 1->2->3->4, 你应该返回 2->1->4->3.

思路：递归,战胜100%，时间 0ms

递归公式：
    首先交换两个结点，然后让后一结点的next指向继续递归后的结果。

代码：

public ListNode swapPairs(ListNode head) {
     if(head==null)//递归结束条件1如果链表中结点个数为偶数，当返回最后一个结点的next时直接返回null
         return null;
     ListNode currNode=head;
     ListNode nextNode=head.next;
        if(nextNode==null)//如果结点个数为奇数，那么返回前一个结点。
        {
            return currNode;
        }
        ListNode temp=nextNode.next;
        nextNode.next=currNode;//将currNode的后一个结点的next设为前一个结点就完成了交换结点的前步。
        currNode.next=swapPairs(temp);//将前一个结点的next设置为后一个结点的next交换后的变换后的结果
        return nextNode;//返回nextNode而非currNode.
    }
 public static void main(String[] args) {
        ListNode node=new ListNode(1);
        node.next=new ListNode(2);
        node.next.next=new ListNode(3);
        node.next.next.next=new ListNode(4);
        node.next.next.next.next=null;
        ListNode res=new o0024().swapPairs(node);
        while(res!=null) {
            System.out.println(res.val);
            res=res.next;
        }
    }

LeetCode-23合并多个链表

2019-11-14
LeetCode

题目：

合并 k 个排序链表，返回合并后的排序链表。请分析和描述算法的复杂度。
示例:
输入: [   1->4->5,   1->3->4,   2->6 ] 输出: 1->1->2->3->4->4->5->6

思路一（自写）：递归：效率极低，战胜15%，运行时间：340ms

每次取出lists的所有链表的第一个元素，找出最小的那个，然后用最小值的那个结点，指着这个结点的指针指向他所在结点后的链表与其他结点继续此操作的返回值。

代码：

public ListNode mergeKLists(ListNode[] lists) {
    int minval=Integer.MAX_VALUE;
    int minIndex=0;
    ListNode head=null;
    for(int i=0;i<lists.length;i++) {
        if(lists[i]==null) {
            continue;
        }
        if(lists[i].val<=minval) {
            minval=lists[i].val;
            minIndex=i;
        }
    }
    if(minval==Integer.MAX_VALUE)
        return null;
    head=lists[minIndex];
    lists[minIndex]=lists[minIndex].next;
    head.next=mergeKLists(lists);
    return head;
}

思路二：归并：效率高，战胜100%，运行时间：2ms

将lists不断合并左右元素，然后放入左半边lists，然后继续在左半边继续合并左右元素操作，直到right到达0处。两个链表的合并算法自己想。

代码：

public ListNode 自写mergeKLists(ListNode []lists) {
    if (lists==null||lists.length<=0)
        return null;
    ListNode head=new ListNode(0);
    int left=0;
    int right=lists.length-1;
    while(right>0) {
        while(left<right) {
            head.next=自写merge(lists[left],lists[right]);
            left++;
            right--;
        }
        left=0;
    }
    return head.next;
}
private ListNode 自写merge(ListNode l1, ListNode l2) {
    if(l1==null)//如果有一个为null，直接返回另一个。
        return l2;
    if(l2==null)
        return l1;
    ListNode head=new ListNode(0);//建一个头结点不做返回值一部分
    ListNode currNode=head;//定义一个零时工。
    while(l1!=null&l2!=null) {
        if(l1.val<l2.val) {
            currNode.next=l1;
            l1=l1.next;
        }else {
            currNode.next=l2;
            l2=l2.next;
        }
        currNode=currNode.next;
    }
    currNode.next=l1==null?l2:l1;//如果有链表没有结束，直接将其加到后面。
    return head.next;
}

LeetCode-22括号生成

LeetCode

题目：

给出 n 代表生成括号的对数，请你写出一个函数，使其能够生成所有可能的并且有效的括号组合。

例如，给出 n = 3，生成结果为：

[
  "((()))",
  "(()())",
  "(())()",
  "()(())",
  "()()()"
]

思路：回溯算法。
直接搬运他人的思路吧。看完之后，我是接连卧槽。

方法一：回溯算法（深度优先遍历）
如果完成一件事情有很多种方法，并且每一种方法分成若干步骤，那多半就可以使用“回溯”算法完成。

“回溯”算法的基本思想是“尝试搜索”，一条路如果走不通（不能得到想要的结果），就回到上一个“路口”，尝试走另一条路。

因此，“回溯”算法的时间复杂度一般不低。如果能提前分析出，走这一条路并不能得到想要的结果，可以跳过这个分支，这一步操作叫“剪枝”。

做“回溯”算法问题的基本套路是：

1、使用题目中给出的示例，画树形结构图，以便分析出递归结构；

一般来说，树形图不用画完，就能够分析出递归结构和解题思路。

2、分析一个结点可以产生枝叶的条件、递归到哪里终止、是否可以剪枝、符合题意的结果在什么地方出现（可能在叶子结点，也可能在中间的结点）；

3、完成以上两步以后，就要编写代码实现上述分析的过程，使用代码在画出的树形结构上搜索符合题意的结果。

在树形结构上搜索结果集，使用的方法是执行一次“深度优先遍历”。在遍历的过程中，可能需要使用“状态变量”。

（“广度优先遍历”当然也是可以的，请参考方法二。）

我们以 n = 2 为例，画树形结构图。

画图以后，可以分析出的结论：

左右都有可以使用的括号数量，即严格大于 00 的时候，才产生分支；

左边不受右边的限制，它只受自己的约束；

右边除了自己的限制以外，还收到左边的限制，即：右边剩余可以使用的括号数量一定得在严格大于左边剩余的数量的时候，才可以“节外生枝”；

在左边和右边剩余的括号数都等于 00 的时候结算。

class Solution {
    public List<String> generateParenthesis(int n) {
         List<String>list=new ArrayList<String>();
         if(n==0)
             return list;
         String str="";
         def(n,n,str,list);
         return list;
        }
    public static void def(int left,int right,String str,List<String>list) {
        if(left==0&&right==0) {
            list.add(str);
            return ;
        }
        if(left>0)
        def(left-1,right,str+"(",list);
        if(right>0&&right>left)
        def(left,right-1,str+")",list);
        return ;
    }
}
