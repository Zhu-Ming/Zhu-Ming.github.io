LeetCode-17电话号码的字母组合

2019-11-08

题目：

给定一个仅包含数字 2-9 的字符串，返回所有它能表示的字母组合。

给出数字到字母的映射如下（与电话按键相同）。注意 1 不对应任何字母。

示例:

输入："23"
输出：["ad", "ae", "af", "bd", "be", "bf", "cd", "ce", "cf"].
说明:
尽管上面的答案是按字典序排列的，但是你可以任意选择答案输出的顺序。

来源：力扣（LeetCode）
链接：https://leetcode-cn.com/problems/letter-combinations-of-a-phone-number
著作权归领扣网络所有。商业转载请联系官方授权，非商业转载请注明出处。

思路：我的思路，每次排序都可以看为将上一次的结果与下一次的要找的字符进行排序，注意将上一次产生的每个结果看成一个整体，然后与后面的字符进行排列，比如“23”，情况下可以看成（a,b,c）与（d,e,f）进行排列，而“233”可以先看成（a,,b,c）与(d,e,f)进行排列，然后这一次排列的结果为（ad, ae, af, bd, be, bf, cd, ce, cf）在与（d,e,f）进行排列。

LeetCode-16最接近的三数之和

2019-11-07

题目：

给定一个包括 n 个整数的数组 nums 和 一个目标值 target。找
出 nums 中的三个整数，使得它们的和与 target 最接近。返回这三
个数的和。假定每组输入只存在唯一答案。

例如，给定数组 nums = [-1，2，1，-4], 和 target = 1.

与 target 最接近的三个数的和为 2. (-1 + 2 + 1 = 2).

来源：力扣（LeetCode）
链接：https://leetcode-cn.com/problems/3sum-closest
著作权归领扣网络所有。商业转载请联系官方授权，非商业转载请注明出处。

思路：这题我是没有想法的，好不容易写的，调试半天，结果竟然位列12%左右，那就直接上大佬的代码；

分析思路：

然后依据上一题的双指针算法，对数组排一下序，从当前位置出发，
记录下每三个数字的和，然后用它们的和与初始的和对比。如果比初始更接近target，就修改要作为返回结果的和。如果当前的和比target大说明减小一点值可能跟接近target，如果比target小，那么可能大一点的和会更接近target。注意此处要进行下一次的循环时
可以优化一下，跳过那些已经出现过的数值。

代码：

class Solution {
    public int threeSumClosest(int[] nums, int target) {
        Arrays.sort(nums);
        int begin,end,currSum,sum=nums[0]+nums[1]+nums[2];
        for(int i=0;i<nums.length-2;i++){
            begin=i+1;
            end=nums.length-1;
            while(begin<end){
            currSum=nums[begin]+nums[end]+nums[i];
            if(Math.abs(target-currSum)<Math.abs(target-sum)){
                sum=currSum;
                }
            if(currSum>target){//如果当前和小于target，那么跳过重复的部分。
                while(begin<end&&nums[end]==nums[end-1])end--;
                end--;
            }else if(currSum<target){//如果当前大于目标值，那么跳过重复的数字，继续下一次循环
                while(begin<end&&nums[begin]==nums[begin+1])begin++;
                begin++;
            }else{
                    return sum;//最接近了，直接返回
                }
            }    
        }
    return sum;
    }
}

LeetCode-15三数之和

2019-11-06

题目：

给定一个包含 n 个整数的数组 nums，判断 nums 中是否存在三个元素 a，b，c ，使得 a + b + c = 0 ？找出所有满足条件且不重复的三元组。

注意：答案中不可以包含重复的三元组。

例如, 给定数组 nums = [-1, 0, 1, 2, -1, -4]，

满足要求的三元组集合为：
[
  [-1, 0, 1],
  [-1, -1, 2]
]

思路：

最优解法：

先对数组排序，从左向右遍历，并将值存储到一个变量中，然后采用双指针，分别从当前值的位置+1处开始，
以及从数组为开始相向而行，每次让两个指针指向的值与当前值相加如果等于0，说明可以存入数组列表中，如
果小于，说明左边指针指向的值过于小了，将其加+1，继续下一的循环，如果大于就说明右边的值过大了，将
将右边的指针指向前一个，继续循环，如果相等，那么要注意：如果两个指针指向下一个位置的值任然为上一
个位置一定要让指针跳出具有相同数字的区域，然后继续下一次循环。

代码：

public List<List<Integer>> threeSum(int[] nums) {
     List<List<Integer>>lists=new ArrayList<>();
     if(nums.length<3) {
         return lists;
     }
        Arrays.sort(nums);
        int currValue=0,a=0,b=0;
        for(int i=0;i<nums.length;i++) {
            currValue=nums[i];
            if(currValue>0) {
                break;
            }
            if(i>0&&nums[i]==nums[i-1]) {
                continue;
            }
            a=i+1;
            b=nums.length-1;
            while(a<b) {
                if(nums[a]+nums[b]==0-currValue) {
                    lists.add(Arrays.asList(currValue,nums[a],nums[b]));
                    while(b>a&&nums[a]==nums[a+1])a++;//跳到在数组中与它等值的最后一个数字处
                    while(b>a&&nums[b]==nums[b-1])b--;
                    a++;//跳向下一个数字处
                    b--;//跳向前一个数字处
                }else if(nums[a]+nums[b]>0-currValue) {
                    b--;
                }else {
                    a++;
                }
            }
        }

            return lists;
    }

LeetCode-14最长公共前缀

2019-11-06

题目：

编写一个函数来查找字符串数组中的最长公共前缀。

如果不存在公共前缀，返回空字符串 ""。

示例 1:

输入: ["flower","flow","flight"]
输出: "fl"
示例 2:

输入: ["dog","racecar","car"]
输出: ""
解释: 输入不存在公共前缀。
说明:

所有输入只包含小写字母 a-z 。

思路:

最优解法：

将一个字符串设置为默认的公共前缀，然后遍历剩余的字符串，查看每一个字符串是否是以此字符串开头的，
如果有一个字符串不是就修改默认的公共前缀的值去掉最后一个字符，再次重复的与上一次不符合的字符串比
对，如果还是不符合，就继续去除最后一个字符直到符合为止，如果都去掉了就直接返回空字符串，如果符合
就再与下一个字符串比对，直到都符合为止。

代码:

public String longestCommonPrefix(String[] strings) {
     if(strings.length == 0){
        return "";
    }
    String prefix = strings[0];
    for (int i=1;i<strings.length;i++){//第几位
        while (strings[i].indexOf(prefix) !=0){
            prefix = prefix.substring(0,prefix.length()-1);
            if (prefix.isEmpty()){
                return "";
            }
        }
    }
    return prefix;
}

我的解法：

思路：公共字符前缀必定时每个字符串的开头相同的部分，只要选定一个字符串，然后不断遍历它的字符，先
将字符加到结果字符串上，然后接着用字符去与其它字符串的对应位置的字符比对，如果当前字符对应的位置
恰好等于某个字符串的长度，那么就返回结果的除最后一个字符外的全部字符，如果其他字符串对应的位置的
字符恰好就是这个，那么证明没有加错字符就继续遍历，如果其他字符串对应位置的字符不是这个说明加错
了，就得返回结果字符串除最后一个字符外的全部字符。如果正常退出循环，就直接返回结果；

代码：

public String longestCommonPrefix(String[] strs) {
    if(strs==null||strs.length<1)
           return "";
       char ch='\0';
       String re="";
           for(int i=0;i<strs[0].length();i++) {
            ch=strs[0].charAt(i);
            re=re+ch;
            for(int j=1;j<strs.length;j++) {
                if(strs[j].length()==i) {
                    return re.substring(0, re.length()-1);
                }
                if(ch==strs[j].charAt(i)) {
                    continue;
                }else {
                    return re.substring(0, re.length()-1);
                }
            }
        }
        return re;
}

LeetCode-13罗马数字转整数

2019-11-06

题目：

罗马数字包含以下七种字符: I， V， X， L，C，D 和 M。

字符          数值
I             1
V             5
X             10
L             50
C             100
D             500
M             1000

例如， 罗马数字 2 写做 II ，即为两个并列的 1。12 写做 XII ，即为 X + II 。 27 写做  XXVII, 即
为 XX + V + II 。

通常情况下，罗马数字中小的数字在大的数字的右边。但也存在特例，例如 4 不写做 IIII，而是 IV。数字 1
在数字 5 的左边，所表示的数等于大数 5 减小数 1 得到的数值 4 。同样地，数字 9 表示为 IX。这个特
殊的规则只适用于以下六种情况：

I 可以放在 V (5) 和 X (10) 的左边，来表示 4 和 9。
X 可以放在 L (50) 和 C (100) 的左边，来表示 40 和 90。 
C 可以放在 D (500) 和 M (1000) 的左边，来表示 400 和 900。
给定一个罗马数字，将其转换成整数。输入确保在 1 到 3999 的范围内。

示例 1:

输入: "III"
输出: 3
示例 2:

输入: "IV"
输出: 4
示例 3:

输入: "IX"
输出: 9
示例 4:

输入: "LVIII"
输出: 58
解释: L = 50, V= 5, III = 3.
示例 5:

输入: "MCMXCIV"
输出: 1994
解释: M = 1000, CM = 900, XC = 90, IV = 4.

最优解法：

依据输入的字符串取出每一个字符，如果如果不是
一些特殊的字符（比如当前字符为C后面的字符为
D），直接让返回值加上这个字符对应的数值，否
则就加上特殊字符串对应的数值（此处应该修改指
定字符位置的数值）。

我的解法：

依照上一题的思路，定义一个字符串数组存储罗马字符，再定义一个数组存储罗马字符串对应的数值，然后从
头开始，匹配字符串遇到合适的就不断地匹配它，然后修改原字符串使这个位置的罗马字符串丢掉。

代码：

public int romanToInt(String s) {
     String[]romans=new String[] {"M","CM","D","CD","C","XC","L","XL","X","IX","V","IV","I"};
     int res[]=new int[] {1000,900,500,400,100,90,50,40,10,9,5,4,1};
     int result=0;
     int i=0;
     while(i<13) {//数组中的字符串只有13个
         while(s.startsWith(romans[i])) {
             s=s.substring(romans[i].length());
             result=result+res[i];
         }
         i++;
     }
     return result;
}

LeetCode-12整数转换罗马数字

2019-11-03

题目：

罗马数字包含以下七种字符： I， V， X， L，C，D 和 M。

字符          数值
I             1
V             5
X             10
L             50
C             100
D             500
M             1000
例如， 罗马数字 2 写做 II ，即为两个并列的 1。12 写做 XII ，即为 X + II 。 27 写做  XXVII, 即为 XX + V + II 。

通常情况下，罗马数字中小的数字在大的数字的右边。但也存在特例，例如 4 不写做 IIII，而是 IV。数字 1
 在数字 5 的左边，所表示的数等于大数 5 减小数 1 得到的数值 4 。同样地，数字 9 表示为 IX。这个特殊的规则只适用于以下六种情况：

I 可以放在 V (5) 和 X (10) 的左边，来表示 4 和 9。
X 可以放在 L (50) 和 C (100) 的左边，来表示 40 和 90。 
C 可以放在 D (500) 和 M (1000) 的左边，来表示 400 和 900。
给定一个整数，将其转为罗马数字。输入确保在 1 到 3999 的范围内。

示例 1:

输入: 3
输出: "III"
示例 2:

输入: 4
输出: "IV"
示例 3:

输入: 9
输出: "IX"
示例 4:

输入: 58
输出: "LVIII"
解释: L = 50, V = 5, III = 3.
示例 5:

输入: 1994
输出: "MCMXCIV"
解释: M = 1000, CM = 900, XC = 90, IV = 4.

思路一：定义一个函数根据每次取余的结果推送相应位数的字符，然后根据余数构建字符串。

代码：

public String intToRoman(int num) {
        String res="";
        int r=0;
        int time=0;
        while(num!=0) {
            time++;
            r=num%10;
            num/=10;
            if(time==1) {
                res=add(res,'I','V','X',r);
            }else if(time == 2) {
                res=add(res,'X','L','C',r);
            }else if(time==3) {
                res=add(res,'C','D','M',r);
            }else {
                res=add(res,'M','\0','\0',r);
            }
        }
        return res;
    }

思路二：

贪心算法，来自百度词条的解释（贪心算法（又称贪婪算法）是指，在对问题求解时，总是做出在当前

来是最好的选择。也就是说，不从整体最优上加以考虑，他所做出的是在某种意义上的局部最优解。

贪心算法不是对所有问题都能得到整体最优解，关键是贪心策略的选择，选择的贪心策略必须具备无后效性，

即某个状态以前的过程不会影响以后的状态，只与当前状态有关。）每一步都使用当前较大的罗马数字作为加法

因子，最后得到罗马数字表示就是长度最少的。做法：将所有特殊的数值以及它们对应的字符串存入数组，然后

根据输入的数值，从高位开始减，一个一个匹配，加入到字符串中。

代码：

public String intToRoman(int num) {
    StringBuilder res=new StringBuilder();
     int nums[]=new int[] {1000,900,500,400,100,90,50,40,10,9,5,4,1};
     String[]romans=new String[] {"M","CM","D","CD","C","XC","L","XL","X","IX","V","IV","I"};
     int index=0;
     while(index<13) {
         while(num>=nums[index]) {
             num=num-nums[index];
             res.append(romans[index]);
         }
         index++;
     }
     return res.toString();
}
