LeetCode-15三数之和

2019-11-06
LeetCode

题目：

给定一个包含 n 个整数的数组 nums，判断 nums 中是否存在三个元素 a，b，c ，使得 a + b + c = 0 ？找出所有满足条件且不重复的三元组。

注意：答案中不可以包含重复的三元组。

例如, 给定数组 nums = [-1, 0, 1, 2, -1, -4]，

满足要求的三元组集合为：
[
  [-1, 0, 1],
  [-1, -1, 2]
]

思路：

最优解法：

先对数组排序，从左向右遍历，并将值存储到一个变量中，然后采用双指针，分别从当前值的位置+1处开始，
以及从数组为开始相向而行，每次让两个指针指向的值与当前值相加如果等于0，说明可以存入数组列表中，如
果小于，说明左边指针指向的值过于小了，将其加+1，继续下一的循环，如果大于就说明右边的值过大了，将
将右边的指针指向前一个，继续循环，如果相等，那么要注意：如果两个指针指向下一个位置的值任然为上一
个位置一定要让指针跳出具有相同数字的区域，然后继续下一次循环。

代码：

public List<List<Integer>> threeSum(int[] nums) {
     List<List<Integer>>lists=new ArrayList<>();
     if(nums.length<3) {
         return lists;
     }
        Arrays.sort(nums);
        int currValue=0,a=0,b=0;
        for(int i=0;i<nums.length;i++) {
            currValue=nums[i];
            if(currValue>0) {
                break;
            }
            if(i>0&&nums[i]==nums[i-1]) {
                continue;
            }
            a=i+1;
            b=nums.length-1;
            while(a<b) {
                if(nums[a]+nums[b]==0-currValue) {
                    lists.add(Arrays.asList(currValue,nums[a],nums[b]));
                    while(b>a&&nums[a]==nums[a+1])a++;//跳到在数组中与它等值的最后一个数字处
                    while(b>a&&nums[b]==nums[b-1])b--;
                    a++;//跳向下一个数字处
                    b--;//跳向前一个数字处
                }else if(nums[a]+nums[b]>0-currValue) {
                    b--;
                }else {
                    a++;
                }
            }
        }

            return lists;
    }

LeetCode-14最长公共前缀

2019-11-06
LeetCode

题目：

编写一个函数来查找字符串数组中的最长公共前缀。

如果不存在公共前缀，返回空字符串 ""。

示例 1:

输入: ["flower","flow","flight"]
输出: "fl"
示例 2:

输入: ["dog","racecar","car"]
输出: ""
解释: 输入不存在公共前缀。
说明:

所有输入只包含小写字母 a-z 。

思路:

最优解法：

将一个字符串设置为默认的公共前缀，然后遍历剩余的字符串，查看每一个字符串是否是以此字符串开头的，
如果有一个字符串不是就修改默认的公共前缀的值去掉最后一个字符，再次重复的与上一次不符合的字符串比
对，如果还是不符合，就继续去除最后一个字符直到符合为止，如果都去掉了就直接返回空字符串，如果符合
就再与下一个字符串比对，直到都符合为止。

代码:

public String longestCommonPrefix(String[] strings) {
     if(strings.length == 0){
        return "";
    }
    String prefix = strings[0];
    for (int i=1;i<strings.length;i++){//第几位
        while (strings[i].indexOf(prefix) !=0){
            prefix = prefix.substring(0,prefix.length()-1);
            if (prefix.isEmpty()){
                return "";
            }
        }
    }
    return prefix;
}

我的解法：

思路：公共字符前缀必定时每个字符串的开头相同的部分，只要选定一个字符串，然后不断遍历它的字符，先
将字符加到结果字符串上，然后接着用字符去与其它字符串的对应位置的字符比对，如果当前字符对应的位置
恰好等于某个字符串的长度，那么就返回结果的除最后一个字符外的全部字符，如果其他字符串对应的位置的
字符恰好就是这个，那么证明没有加错字符就继续遍历，如果其他字符串对应位置的字符不是这个说明加错
了，就得返回结果字符串除最后一个字符外的全部字符。如果正常退出循环，就直接返回结果；

代码：

public String longestCommonPrefix(String[] strs) {
    if(strs==null||strs.length<1)
           return "";
       char ch='\0';
       String re="";
           for(int i=0;i<strs[0].length();i++) {
            ch=strs[0].charAt(i);
            re=re+ch;
            for(int j=1;j<strs.length;j++) {
                if(strs[j].length()==i) {
                    return re.substring(0, re.length()-1);
                }
                if(ch==strs[j].charAt(i)) {
                    continue;
                }else {
                    return re.substring(0, re.length()-1);
                }
            }
        }
        return re;
}

LeetCode-13罗马数字转整数

2019-11-06
LeetCode

题目：

罗马数字包含以下七种字符: I， V， X， L，C，D 和 M。

字符          数值
I             1
V             5
X             10
L             50
C             100
D             500
M             1000

例如， 罗马数字 2 写做 II ，即为两个并列的 1。12 写做 XII ，即为 X + II 。 27 写做  XXVII, 即
为 XX + V + II 。

通常情况下，罗马数字中小的数字在大的数字的右边。但也存在特例，例如 4 不写做 IIII，而是 IV。数字 1
在数字 5 的左边，所表示的数等于大数 5 减小数 1 得到的数值 4 。同样地，数字 9 表示为 IX。这个特
殊的规则只适用于以下六种情况：

I 可以放在 V (5) 和 X (10) 的左边，来表示 4 和 9。
X 可以放在 L (50) 和 C (100) 的左边，来表示 40 和 90。 
C 可以放在 D (500) 和 M (1000) 的左边，来表示 400 和 900。
给定一个罗马数字，将其转换成整数。输入确保在 1 到 3999 的范围内。

示例 1:

输入: "III"
输出: 3
示例 2:

输入: "IV"
输出: 4
示例 3:

输入: "IX"
输出: 9
示例 4:

输入: "LVIII"
输出: 58
解释: L = 50, V= 5, III = 3.
示例 5:

输入: "MCMXCIV"
输出: 1994
解释: M = 1000, CM = 900, XC = 90, IV = 4.

最优解法：

依据输入的字符串取出每一个字符，如果如果不是
一些特殊的字符（比如当前字符为C后面的字符为
D），直接让返回值加上这个字符对应的数值，否
则就加上特殊字符串对应的数值（此处应该修改指
定字符位置的数值）。

我的解法：

依照上一题的思路，定义一个字符串数组存储罗马字符，再定义一个数组存储罗马字符串对应的数值，然后从
头开始，匹配字符串遇到合适的就不断地匹配它，然后修改原字符串使这个位置的罗马字符串丢掉。

代码：

public int romanToInt(String s) {
     String[]romans=new String[] {"M","CM","D","CD","C","XC","L","XL","X","IX","V","IV","I"};
     int res[]=new int[] {1000,900,500,400,100,90,50,40,10,9,5,4,1};
     int result=0;
     int i=0;
     while(i<13) {//数组中的字符串只有13个
         while(s.startsWith(romans[i])) {
             s=s.substring(romans[i].length());
             result=result+res[i];
         }
         i++;
     }
     return result;
}
