LeetCode-12整数转换罗马数字

2019-11-03
LeetCode

题目：

罗马数字包含以下七种字符： I， V， X， L，C，D 和 M。

字符          数值
I             1
V             5
X             10
L             50
C             100
D             500
M             1000
例如， 罗马数字 2 写做 II ，即为两个并列的 1。12 写做 XII ，即为 X + II 。 27 写做  XXVII, 即为 XX + V + II 。

通常情况下，罗马数字中小的数字在大的数字的右边。但也存在特例，例如 4 不写做 IIII，而是 IV。数字 1
 在数字 5 的左边，所表示的数等于大数 5 减小数 1 得到的数值 4 。同样地，数字 9 表示为 IX。这个特殊的规则只适用于以下六种情况：

I 可以放在 V (5) 和 X (10) 的左边，来表示 4 和 9。
X 可以放在 L (50) 和 C (100) 的左边，来表示 40 和 90。 
C 可以放在 D (500) 和 M (1000) 的左边，来表示 400 和 900。
给定一个整数，将其转为罗马数字。输入确保在 1 到 3999 的范围内。

示例 1:

输入: 3
输出: "III"
示例 2:

输入: 4
输出: "IV"
示例 3:

输入: 9
输出: "IX"
示例 4:

输入: 58
输出: "LVIII"
解释: L = 50, V = 5, III = 3.
示例 5:

输入: 1994
输出: "MCMXCIV"
解释: M = 1000, CM = 900, XC = 90, IV = 4.

思路一：定义一个函数根据每次取余的结果推送相应位数的字符，然后根据余数构建字符串。

代码：

public String intToRoman(int num) {
        String res="";
        int r=0;
        int time=0;
        while(num!=0) {
            time++;
            r=num%10;
            num/=10;
            if(time==1) {
                res=add(res,'I','V','X',r);
            }else if(time == 2) {
                res=add(res,'X','L','C',r);
            }else if(time==3) {
                res=add(res,'C','D','M',r);
            }else {
                res=add(res,'M','\0','\0',r);
            }
        }
        return res;
    }
 public static String add(String str,char ch1,char ch2,char ch3,int r) {

思路二：

贪心算法，来自百度词条的解释（贪心算法（又称贪婪算法）是指，在对问题求解时，总是做出在当前

来是最好的选择。也就是说，不从整体最优上加以考虑，他所做出的是在某种意义上的局部最优解。

贪心算法不是对所有问题都能得到整体最优解，关键是贪心策略的选择，选择的贪心策略必须具备无后效性，

即某个状态以前的过程不会影响以后的状态，只与当前状态有关。）每一步都使用当前较大的罗马数字作为加法

因子，最后得到罗马数字表示就是长度最少的。做法：将所有特殊的数值以及它们对应的字符串存入数组，然后

根据输入的数值，从高位开始减，一个一个匹配，加入到字符串中。

代码：

public String intToRoman(int num) {
    StringBuilder res=new StringBuilder();
     int nums[]=new int[] {1000,900,500,400,100,90,50,40,10,9,5,4,1};
     String[]romans=new String[] {"M","CM","D","CD","C","XC","L","XL","X","IX","V","IV","I"};
     int index=0;
     while(index<13) {
         while(num>=nums[index]) {
             num=num-nums[index];
             res.append(romans[index]);
         }
         index++;
     }
     return res.toString();
}

LeetCode-11盛水最多的容器

2019-11-03
LeetCode

题目：

给定 n 个非负整数 a1，a2，...，an，每个数代表坐标中的一个点 (i, ai) 。在坐标内画 n 条垂直线，垂

直线 i 的两个端点分别为 (i, ai) 和 (i, 0)。找出其中的两条线，使得它们与 x 轴共同构成的容器可以

容纳最多的水。

说明：你不能倾斜容器，且 n 的值至少为 2。

图中垂直线代表输入数组 [1,8,6,2,5,4,8,3,7]。在此情况下，容器能够容纳水（表示为蓝色部分）的最大

值为 49。


示例:

输入: [1,8,6,2,5,4,8,3,7]

输出: 49

下图是此数组对应的图形，以及它的最大面积。

来源：力扣（LeetCode）
链接：https://leetcode-cn.com/problems/container-with-most-water

##思路1：
###暴力解法（优化后的）

选定长方形的长，然后依据长来从头开始找数组中的相距长度为此的两个数值，取出它们的最小值，然后求面

积，如果面积比上一个值大，就修改上一个面积值，使得面积永远是最大值。

代码块：

public int maxArea(int[] height) {
    int area=0;
    int h=0;
    for(int i=1;i<height.length;i++) {//根据不同的长来查找
        for(int a=0;a<height.length;a++) {
            int b=a+i;
            if(b>height.length-1) {
                break;
            }else {
                h=height[a]>height[b]?height[b]:height[a];
                if(h*i>area) {
                    area=h*i;
                }
            }
        }

    }
    return area;
}

复杂度分析

时间复杂度：O(n^2)，计算所有 n(n−1)/2种高度组合的面积。

空间复杂度：O(1)，使用恒定的额外空间。

##思路二
###双指针法（最优解法）

首先，定义两个变量分别指向数组的头和尾下标，然后比较它们对应的数组中的值，取小的数值与它们之间的
距离相乘，这就是当前的面积，如果当前面积比上一个面积大就修改面积变量值，使其永远存放最大面积值。

####分析双指针法：

因为此思路是根据长方形的面积始终由它的最小的宽决定的，所以从两头开始遍历，改变指向小数值的那个变
量使其指向下一个（对于前面的来说是后一个，对于后面的来说是前一个），（为什么这么做呢?因为改变了小
的那个变量的数值使两个变量之间对应的数组值中即使有比它还要小的，那么这个值与其他数组值形成的面积
也绝对不会比上一个大）

代码块：

public int maxArea(int[] height) {
    int area=0;
    int head=0,last=height.length-1;
    int h=0;
        while(head<last) {
            h=height[head]>height[last]?height[last--]:height[head++];
            if(h*(last-head+1)>area) {
                area=h*(last-head+1);
            }
        }
    return area;
}

时间复杂度：O(n)

空间复杂度：O(1)

LeetCode-9回文数

2019-11-02
LeetCode

题目：

判断一个整数是否是回文数。回文数是指正序（从左向右）和倒序（从右向左）读都是一样的整数。

示例 1:

输入: 121
输出: true
示例 2:

输入: -121
输出: false
解释: 从左向右读, 为 -121 。 从右向左读, 为 121- 。因此它不是一个回文数。
示例 3:

输入: 10
输出: false
解释: 从右向左读, 为 01 。因此它不是一个回文数。

最优解法：将首先可以肯定数字为负数或者（末尾有0且不等于0）情况下其反转后一定是错的。0一定是对的。下面讨论数为正数，（正数的长度为偶数）只要正数的前一半的数值等于后一半的数值就可以了（正数的长度为奇数的话）只要后一半的数值等于前一半对10取整即可（因为中间的那一个不需要有对称的位置）。不过我这里将其改了一下使代码更加简单（运行时间与原代码相差不大）

代码：

public boolean isPalindrome(int x) {
    if (x < 0 || (x % 10 == 0 && x != 0)) return false;
        int reverseNum=0;
        int y=x;
        while(x!=0) {
            reverseNum=reverseNum*10+x%10;
            x=x/10;
        }
        return y==reverseNum;
}

我的解法：将一个字符串，数字小于0返回false，为0返回true，对于正数将他赋值到一个字符串里，然后分别定义两个变量从头，尾开始向中间遍历，如果他们所在位置的字符不同说明不对称（即无法成为回文结构）。

代码：

public boolean isPalindrome(int x) {
    if(x<0)
        return false;
    if(x==0)
        return true;
        String str=""+x;
        int i=0,j=str.length()-1;
        while(i<j) {
            if(str.charAt(i)!=str.charAt(j)) {
                return false;
            }else {
                i++;
                j--;
            }
        }
        return true;
}

总结：字符串的操作运行时间要多余直接对数值进行操作的时间，应尽量采用操作数值。
